# 0000: <title>

## Summary

One paragraph, max five sentences. What does this change, and why.

## Motivation

What problem does this solve? Who feels it? Cite real code or real
users wherever possible; speculative pain is the weakest form of
motivation.

## Guide-level explanation

Teach this feature to someone who already knows XS. Write it the way
it will appear in the docs. Include at least two examples that cover
the happy path.

## Reference-level explanation

The precise rules. For grammar changes, show the new production. For
type system changes, show the inference / checking rules. For stdlib,
list the full public surface with signatures and guaranteed semantics.

## Drawbacks

Every design has costs. Name them honestly. If you can't think of any,
you haven't thought about it long enough.

## Rationale and alternatives

What other designs were considered? Why this one? What is the smallest
plausible version of this feature?

## Prior art

Which other languages have shipped something similar? What did they
learn? Don't skip this.

## Unresolved questions

What do you still not know? What needs user feedback to decide?

## Future possibilities

Natural extensions that this RFC enables but does not propose.

## Impact on stability

Which tier does this land in? Is it additive, or does it break Tier 1
code? If breaking, what's the migration path?
